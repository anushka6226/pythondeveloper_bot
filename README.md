Binance Futures Trading Bot (Mock Implementation)
Project Overview
This project is a Python-based trading bot designed to interface with the Binance Futures API.

Architectural Note: Due to synchronization and permission issues with the Binance Testnet API key generation, this implementation utilizes a Mock Object Pattern. This allows the bot to simulate full order execution and account management without requiring active API keys, ensuring the logic is production-ready and CI/CD compatible.

Features
Mock Binance Client: A custom-built class that mirrors the python-binance library, simulating futures_create_order and futures_account_balance.

Automated Logging: Integrated Python logging module that records every trade (Symbol, Side, Quantity, Price) into a local bot.log file.

Environment Security: Configured to use Google Colab Secrets (userdata) for API key management, preventing sensitive data leaks.

Scalability: The code structure allows for an instant switch to live trading by swapping the client initialization.

Technical Stack
Language: Python 3.10+

Environment: Google Colab / Jupyter Notebook

Libraries: python-binance, logging, google.colab

Project Structure
Trading_Bot.ipynb: The main execution logic and bot brain.

bot.log: The execution history generated by the bot.

README.md: Project documentation and setup guide.

How It Works
The bot initializes a MockBinanceClient. When a trade is triggered:

The bot validates the account balance.

The order is processed through the mock execution engine.

A success response is returned in the exact JSON format used by the Binance API.

The transaction is time-stamped and saved to bot.log.
